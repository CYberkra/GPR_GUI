# GUI 参数预设系统 v1（Week2 Day8-Day9）

## 目标与范围
本次仅在 GUI 交互层新增参数预设能力，不改动核心算法实现与语义。预设用于快速切换“速度/稳健/细节”三类参数组合。

## 预设定义

### 1) 快速预览（速度优先）
- 适用场景：大文件初筛、先看结构后精调
- 关键 UI 参数：
  - 启用分块预览：开
  - 最大采样点/道数：256 / 120
  - 显示降采样：开（500 / 260）
  - 归一化/去均值：关/关
  - 百分位拉伸：关
- 关键方法参数：
  - dewow.window=21
  - fk_filter=(16,45,3)
  - sec_gain=(1.0,3.2,1.0)
  - hankel_svd=(48,1)

### 2) 降噪优先（稳健）
- 适用场景：噪声重、目标弱、希望稳定可复现
- 关键 UI 参数：
  - 启用分块预览：关
  - 显示降采样：开（900 / 420）
  - 归一化/去均值：开/开
  - 百分位拉伸：开（1,99）
- 关键方法参数：
  - dewow.window=61
  - fk_filter=(12,55,4)
  - sec_gain=(1.0,4.2,1.2)
  - hankel_svd=(96,2)

### 3) 保细节（细节优先）
- 适用场景：边界/薄层细节观察、后续解释需求高
- 关键 UI 参数：
  - 启用分块预览：关
  - 显示降采样：关
  - 归一化/去均值：关/开
  - 百分位拉伸：开（0.5,99.5）
- 关键方法参数：
  - dewow.window=31
  - fk_filter=(8,62,2)
  - sec_gain=(1.0,5.0,1.05)
  - hankel_svd=(72,3)

## 交互能力落地
- 新增预设选择下拉与“应用预设”按钮（单击应用到当前参数控件）
- 新增“回填当前参数”按钮（将当前方法参数保存并回填 UI）
- 预设应用后会在状态栏显示“已应用预设：xxx”，并写入日志
- 方法参数采用“默认值 + 预设覆盖/用户回填覆盖”的策略，不影响核心算法

## 验证结果

   - 校验快速预览预设后关键控件值与方法参数值正确
2. `test_preset_switch_keeps_pipeline_build_stable_min_regression`
   - 校验预设切换后参数解析稳定，流程关键方法参数仍完整可用

## 风险点
- 预设参数为经验值，可能不适配所有设备/地层数据
- 用户手动修改与预设切换并存时，覆盖顺序需保持一致（当前实现：后应用者优先）

## 回滚点
- 代码级：移除 `GUI_PRESETS_V1` 与相关 UI 按钮/方法（`apply_selected_preset`、`_apply_preset_by_key`、`backfill_current_method_params`）即可回到无预设版本
- 行为级：不使用预设时，参数仍按原默认值工作
